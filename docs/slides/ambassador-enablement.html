The Motion
What customers need
Technical teams know they need AI agents. They struggle to get past prototype — no clear path to measure quality, harden behavior, or prove it works before production.
What this workshop delivers
In 2 hours, participants build a production-grade agent end-to-end using vibe coding on Databricks. They leave with a working app, before-and-after eval scores, and a mental model of the full agent lifecycle. SAs leave with a qualified use case and an active POC.

Pre-Workshop Setup
Run this 1 week before. Idempotent — safe to re-run.
1
Ensure participants have Databricks workspace access and Coda installed
2
Clone the repo: https://github.com/rmosleydb/databricks-agentic-app-workshop
3
Run workspace_setup.py — creates catalog, loads data, builds VS index, provisions Lakebase, writes setup-state.json (30–45 min cold; ~10 min warm)
4
Run user_setup.py per participant — or they self-onboard via Claude during the session
5
Deploy reference/agent/ with databricks bundle deploy — gives you a fallback app URL
6
Pre-flight: open the reference app, ask about a product, confirm it responds

Step 2 — Build
Your job: unblock, not instruct.
Common blockers
UC Function creation fails → tell Claude: “What permission error do you see? Fix it.”
App deploys but chat UI doesn’t load → “Check the app logs.”
Lakebase not connecting → “Check the LAKEBASE_SCHEMA env var in app.yaml.”
Participant writing raw code → “Describe what you want to Claude instead.”
Fallback
If a participant can’t get their app running by end of Step 2, share the reference app URL you deployed in setup. They continue Steps 3–5 on the reference app — same eval experience, nothing lost.
Step 4 — Evaluate
Guide participants through each sub-step. Do not let Claude run this silently.
4a
Generate traces
20 scripted conversations → MLflow experiment. Watch, then open Experiments UI.
4b
Curate eval dataset
Tell Claude to build a labeled dataset from traces. Review in MLflow UI.
4c
Build LLM judges
Participants describe quality issues from Step 3. Claude writes 3–4 Guidelines scorers.
4d
Run eval
Tell Claude: run mlflow.genai.evaluate(). First run = baseline scores.
mlflow.genai.evaluate() requires mlflow>=2.19.0. Check before the session.
